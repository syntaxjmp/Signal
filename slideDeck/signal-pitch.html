09 / Defensibility
A4 · Defensibility & Functionality

Not a chatbot — a grounded security brain

General-purpose LLM
Claude-style tools
  • No durable project memory across scans—each prompt is a one-off.
  • Can hallucinate fixes without your repo, CI, or finding IDs as source of truth.
  • Not integrated into the developer workflow (PRs, audits, baselines).
Signal
Stateful AppSec + workspace
  • Memory graph: baselines, dismissals, regressions, and embeddings (Qdrant) tied to real entities—not a blank slate each time.
  • VS Code extension: brings findings and context into the editor where code actually ships.
  • Resolution agent + PRs: AI fixes land as reviewable merges, not pasted snippets.

Why it matters: competitors and generic LLMs don’t maintain a living memory layer for your codebase—Signal does, so analysis stays consistent, explainable, and defensible for auditors.
